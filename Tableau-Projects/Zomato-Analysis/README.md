#  Zomato Data Fusion & Restaurant Analysis (Tableau)

##  Project Overview
This project analyzes restaurant data from Zomato by combining multiple datasets to uncover insights into restaurant performance, customer preferences, and pricing trends. The goal is to support data-driven decision-making for improving customer experience and business strategy.

##  Tools & Technologies
- Tableau (Data Visualization)
- Data Joining
- Exploratory Data Analysis (EDA)

##  Dataset Description
The project uses two datasets:
1. Location Table (restaurant details such as name, city, cuisine)
2. Rating Table (ratings, votes, and cost information)

Both tables are joined using a common key to create a unified dataset.

##  Key Analysis & Visualizations
- Table: Number of restaurants by city (descending order)  
- Heatmap: Average rating across cities  
- Bar Chart: Average cost by city  
- Chart: Number of branches per restaurant  
- Packed Bubble Chart: Top 10 restaurants across cities  
- Bar Chart: Top 5 restaurants based on rating (min 10 votes)  
- KPI Cards:
  - Total Restaurants  
  - Average Cost  
  - Average Rating  
  - Total Votes  

##  Dashboard Features
- Interactive dashboard with city filter  
- Dashboard actions for dynamic filtering  
- Zomato-themed design and formatting  
- Integrated multiple visualizations  

##  Key Insights
- Identified cities with highest restaurant concentration  
- Analyzed cost and rating variations across cities  
- Highlighted top-performing restaurants  
- Explored customer engagement using votes and ratings  

##  Conclusion
This project demonstrates how data joining and visualization can be used to analyze real-world business data and generate actionable insights for platforms like Zomato.
